# Java

Java作为一门跨平台的高级语言，一次编写，到处运行。Java通过Java虚拟机的形式来屏蔽底层硬件、操作系统实现，向上提供了统一API，进而实现跨平台特性。

开发人员通过高级语言编写java源文件，javac将源代码编译为class字节码文件，JVM解释执行class文件。

要使用Java，需要安装JDK，请参考文章：JDK安装 - 梦云智 (opens new window)。

Java Development Kit：Java开发工具包，包含了执行Java代码的虚拟机、Java的核心类库以及Java编译运行的开发工具。

命令行中，执行java -version，出现类似如下结果，安装成功。

➜  ~ java -version
openjdk version "1.8.0_312"
OpenJDK Runtime Environment (Temurin)(build 1.8.0_312-b07)
OpenJDK 64-Bit Server VM (Temurin)(build 25.312-b07, mixed mode)

# Maven

Apache Maven是常用的Java项目管理工具，通过POM文件可以管理项目的构建、发布。

如果你用过其他语言项目的包管理器，可能更容易理解：Java的Maven，类似于php的compose、Node.js的npm等等。

一个标准的Maven项目结构如下：

.
├── pom.xml              # 项目POM文件
└── src                  # 源代码
    ├── main             # 源代码包
    │   └── java
    └── test             # 单元测试源代码包
        └── java

在没有Maven的年代，项目的依赖都是开发者都是去开源网站上下载一个个jar包，并在IDE中手动引入，非常复杂，感兴趣的可以参考博客：不通过maven，idea中对jar包的管理 (opens new window)

有了Maven，我们不需要手动去管理依赖了，所有的开源java项目，都通过mvn deploy将包发布到Maven 中央仓库 (opens new window)，在进行jar包时，只需要在pom.xml文件中声明对包的依赖。

如果需要使用fastjson，在pom中声明groupId、artifactId、version，可以根据这三个项唯一确定一个jar包，Maven会自动去中央仓库进行包下载，并构建到项目中，如下图：

注：为了避免冲突，一般规范中，groupId为组织域名的反写，artifactId为组织内唯一项目名。

Maven安装，请参考文章：MAVEN安装 - 梦云智 (opens new window)。
